    Automatische Anlage von Lagerhaltungsdaten für Artikelvarianten

    Mithilfe der automatischen Anlage von Lagerhaltungsdaten können Sie die Bestandsverwaltung bei vielen Artikelvarianten optimieren.

    So legen Sie eine Artikelvariante an

    1. Klicken Sie im Menüband Dynamics 365 Business Central auf das Symbol Lupe , geben Sie im Suchfeld Lagerhaltungsdaten ein und klicken Sie auf das übereinstimmende Suchergebnis.
    2. Klicken Sie im Menüband auf die Schaltfläche Neu, um Lagerhaltungsdaten für einen Artikel anzulegen.
    3. Klicken Sie im Inforegister Allgemein in das Feld Artikelnr. und wählen Sie einen Datensatz aus.
    4. Klicken Sie im Inforegister Allgemein in das Feld Lagerortcode und wählen Sie einen Datensatz aus.

    5. Klicken Sie im Menüband auf die Schaltfläche Zugehörig > Artikel > Varianten oder klicken Sie alternativ im Inforegister Allgemein in das Feld Variantencode.

      Hinweis

      Wenn im Feld Lagerortcode kein Datensatz ausgewählt wurde, kann die Seite Artikelvarianten nur über das Feld Variantencode geöffnet werden.

    6. Richten Sie die Artikelvariante über diese Felder ein:

      Feld Beschreibung
      Code Geben Sie einen Code zum Kennzeichnen der Artikelvariante ein.
      Beschreibung Geben Sie eine Kurzbeschreibung für die Artikelvariante ein.
      Gesperrt Setzen Sie einen Haken, wenn der verknüpfte Datensatz zum Buchen in Transaktionen gesperrt ist.
      Verkauf gesperrt Setzen Sie einen Haken, wenn die Artikelvariante in Verkaufsbelegen und Buch.-Blätter nicht ausgewählt werden soll.
      Service gesperrt Setzen Sie einen Haken, wenn die Artikelvariante in Serviceartikeln, Serviceverträgen und Servicebelegen, mit Ausnahme von Gutschriften, nicht ausgewählt werden soll.
      Einkauf gesperrt Setzen Sie einen Haken, wenn die Artikelvariante in Einkaufsbelegen und Buch.-Blätter nicht ausgewählt werden soll.
      GTIN Geben Sie die GTIN, die weltweit eindeutige Identifikationsnummer, für diese Artikelvariante an.
      Ursprüngliche Artikelnr. Geben Sie die Herkunftsartikelnummer an, aus der die Artikelvariante erstellt wurde.
      Hinweis

      Es werden automatisch für diese Artikelvariante die gleichen Lagerhaltungsdaten wie für den Artikel angelegt.
