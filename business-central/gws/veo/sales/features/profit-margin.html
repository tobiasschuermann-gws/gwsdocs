    Deckungsbeitrag

    Der Deckungsbeitrag (DB) ermöglicht die Überprüfung des zeilen- oder gesamtbelegbezogenen Deckungsbeitrags in % während des Verkaufsvorgangs. Dabei werden neben Zeilenrabatten auch Positionsarten wie Artikel, Sachkonto, Ressource, Anlage und Zu-/Abschlag (Artikel) berücksichtigt.

    So hinterlegen Sie einen Mindestdeckungsbeitrag für Artikel

    1. Klicken Sie im Menüband Dynamics 365 Business Central auf das Symbol Lupe , geben Sie im Suchfeld Artikel ein und klicken Sie auf das übereinstimmende Suchergebnis.
    2. Öffnen Sie eine Artikelkarte.
    3. Geben Sie im Inforegister Preise und Verkauf im Feld Mindest-DB % einen Mindestdeckungsbeitrag in Prozent ein.
    Hinweis
    • Den Deckungsbeitrag können Sie sowohl im Inforegister Zeilen im Feld DB % als auch im Summenbereich unterhalb der Zeilen im Feld Gesamt-DB % einsehen. Der Deckungsbeitrag wird automatisch aktualisiert, sobald Sie die aktive Zeile verlassen.
    • Die Felder DB (MW) und DB % zeigen den Deckungsbeitrag in grün an, wenn der Mindestdeckungsbeitrag erreicht wird. Dies gilt, sofern der Mindestdeckungsbeitrag in % des Artikels > 0 ist. Andernfalls werden die Felder in roter Farbe angezeigt, wenn der Mindestdeckungsbeitrag nicht erreicht wird.
    • Da auch Nicht-Artikelzeilen in die DB-Betrachtung einbezogen werden, empfiehlt es sich, den Einstandspreis (zum Beispiel von einkalkulierten Frachtkosten) bei Bedarf manuell im Inforegister Zeilen in das Feld Einstandspreis (MW) einzugeben.
    • In Verkaufsangeboten haben Alternativpositionen keinen Einfluss auf den Deckungsbeitrag.
    • Bei der Archivierung des Belegs werden die Felder DB % und Gesamt-DB % automatisch übernommen.

    So bearbeiten Sie den Deckungsbeitrag in Verkaufsbelegen

    Hier am Beispiel eines Verkaufsangebots:

    1. Klicken Sie im Menüband Dynamics 365 Business Central auf das Symbol Lupe , geben Sie im Suchfeld Verkaufsangebote ein und klicken Sie auf das übereinstimmende Suchergebnis.
    2. Öffnen Sie ein Verkaufsangebot.
    3. Markieren Sie im Inforegister Zeilen einen Artikel.
    4. Geben Sie im Feld DB % einen Wert in Prozent ein.
    5. Bestätigen Sie Ihre Eingabe mit Enter.
    Hinweis
    • Wenn das Feld DB % angepasst wird, erfolgt eine Neuberechnung aller verkaufspreisrelevanten Felder in der Verkaufszeile auf Grundlage des Einstandspreises. Dabei werden Zeilenrabatte und Konditionen für diese Zeile verworfen. Ein eventuell hinterlegter Rechnungsrabatt im Beleg bleibt erhalten und wird neu berechnet. Die Grundlage für diese Berechnung kann entweder das Feld Rechnungsrabattbetrag oder Rechnungsrabatt % Wert sein, abhängig von der Rechnungsrabattberechnung im Beleg.
    • In den Belegen Verkaufsangebot, -auftrag, -rechnung, -gutschrift, -reklamation und Rahmenauftrag kann das Feld DB % manuell angepasst werden.
