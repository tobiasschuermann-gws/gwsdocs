    Eingehende und ausgehende Webanfragen

    So richten Sie den Datentransferkonnektor ein

    Über den Datentransferkonnektor werden alle eingehende sowie ausgehende Webanfragen über eine zentrale Stelle angenommen und an Ihr System weitergeleitet.

    1. Klicken Sie im Menüband Dynamics 365 Business Central auf das Symbol Lupe , geben Sie im Suchfeld Datentransfer Einrichtung ein und klicken Sie auf das übereinstimmende Suchergebnis.
    2. Geben Sie im Inforegister Datentransferkonnektor im Feld URL Ihre zugewiesene URL ein.
    3. Geben Sie im Feld Zugriffsschlüssel den Ihren zugewiesenen Zugriffsschlüssel ein.
    Hinweis

    Die benötigten Daten erfahren Sie durch Ihren Berater.

    So richten Sie eingehende Verbindungen ein

    Über die eingehenden Verbindungen richten Sie eine Schnittstelle für Ihre Partner ein, die Daten an Ihr System senden oder abfragen wollen. Eine eingehende Verbindung wird für einen oder mehreren Partnern genutzt.

    1. Klicken Sie im Menüband Dynamics 365 Business Central auf das Symbol Lupe , geben Sie im Suchfeld Eingehende Verbindungen ein und klicken Sie auf das übereinstimmende Suchergebnis.
    2. Klicken Sie im Menüband auf die Schaltfläche Neu.
    3. Geben Sie Inforegister Allgemein im Feld Code eine Bezeichnung und im Feld Beschreibung eine Kurzbeschreibung ein.
    4. Klicken Sie im Inforegister Zugriff in das Feld Benutzer und wählen Sie einen Benutzer aus, der für die eingehenden Anfragen des Datentransferkonnektors verwendet wird. Die Rechte und Einrichtungen des ausgewählten Benutzers werden hierbei berücksichtigt.

    5. Geben Sie im Feld Zugriffschlüssel den Zugriffsschlüssel des ausgewählten Benutzers ein. Diesen können Sie der Benutzerkarte entnehmen. Gehen Sie hierbei wie folgt vor:

      1. Klicken Sie im Menüband Dynamics 365 Business Central auf das Symbol Lupe , geben Sie im Suchfeld Benutzer ein und klicken Sie auf das übereinstimmende Suchergebnis.
      2. Markieren Sie den Benutzer, den Sie ausgewählt haben und klicken Sie in das Feld Benutzername, um die Benutzerkarte zu öffnen.
      3. Klicken Sie im Inforegister Allgemein, Gruppe Webdienst in das Feld Webdienst-Zugriffsschlüssel, markieren Sie den gesamten Zugriffsschlüssel und kopieren Sie diesen Wert.
      4. Schließen Sie die Seite Benutzerkarte, sowie die Seite Benutzer über die Schaltfläche Zurück.

      5. Klicken Sie in das Feld Zugriffsschlüssel und fügen Sie den kopierten Wert ein.

        Hinweis

        Wenn Sie für den Benutzer noch keinen Schlüssel erzeugt haben, finden Sie weitere Informationen unter Generieren eines Web-Service-Zugriffsschlüssel.

    6. Klicken Sie im Menüband auf die Schaltfläche Verbindung Anmelden. Der Schalter Angemeldet im Inforegister Allgemein wird aktiviert. Ab sofort kann diese Verbindung durch Ihre Partner genutzt werden.
    Tipp

    Wir empfehlen für jeden Partner eine eigene Verbindung und einen eigenen Benutzer zu erstellen. Über den Identifizierungsschlüssel und den Benutzer können Sie nachvollziehen, welcher Partner welche Anfragen gestellt hat.

    So richten Sie Funktionszuordnungen ein

    Nachdem Sie den Datentransferkonnektor für eingehende Webanfragen eingerichtet haben, benötigen Sie nun eine Programmlogik, die die eingehenden Webanfragen verarbeitet.

    1. Klicken Sie im Menüband Dynamics 365 Business Central auf das Symbol Lupe , geben Sie im Suchfeld Eingehende Verbindungen ein und klicken Sie auf das übereinstimmende Suchergebnis.
    2. Markieren Sie eine eingehende Verbindung und klicken Sie im Menüband auf die Schaltfläche Bearbeiten.
    3. Geben Sie im Inforegister Funktionszuordnungen im Feld Funktionsname eine Bezeichnung ein.
    4. Geben Sie im Feld Codeunit ID eine Codeunitnummer ein, die eine Verarbeitung vornehmen kann.
